PART 1

关于 DAO 治理

1.1

链上治理

链上治理是指在区块链网络中进行的去中心化治理过程。它涉及到网络中各参与者就协议参数、软件升级、资源分配等问题达成共识的一种方式。链上治理的目标是实现一个公平、透明且不受单一实体控制的网络。链上治理一般通过Governor 治理合约实现。

Governor 智能合约是一种常用的开源治理合约，允许代币持有者使用完全基于链上的投票来控制 DAO。智能合约可以设计为根据链上投票的结果自动执行提案，从而无需受信任的第三方或核心团队来制定投票结果。提案可以做任何链上的事情：

● 从资金库中发送资金 

● 更新 DeFi 协议的参数

● 更改子 DAO 的权限 

● 铸造 NFT 

● 或修改 Governor 本身的规则

去中心化协议从公开发布之时起就处于不断发展之中。通常，最初的团队在最初阶段保留对这种演变的控制，但最终将其委托给利益相关者社区。这个社区做出决策的过程被称为链上治理，它已成为去中心化协议的核心组成部分，推动各种决策，如参数调整、智能合约升级、与其他协议的集成、财务管理、拨款等。

1.2

链下治理

链下治理（Off-chain governance）是指在区块链网络之外进行的治理过程。它通常涉及到社区成员、项目团队、核心开发者等在非区块链平台上讨论和达成共识的方式。 

链下治理的主要特点和优势包括：

● 更多灵活性：链下治理不受区块链技术限制，能够灵活地进行讨论、协商和决策，便于人工干预和快速响应。 

● 多样的参与者：链下治理的参与方式更为多样，可以包括社区成员、项目团队、核心开发者等不同的利益相关者。 

● 多渠道沟通：链下治理可能涉及到线下沟通、论坛讨论、discord、telegram 等多种沟通方式，有助于收集更多的观点和建议。 

然而，链下治理也存在一些缺点： 

● 透明度较低：链下治理过程可能不够透明，难以实时追踪和审计。 

● 容易受中心化影响：链下治理的决策过程容易受到项目团队、核心开发者等中心化实体的影响，可能导致权力滥用和利益输送。 

● 可能不按提案和共识执行治理决策：链下治理的决策结果需要人工实施，执行过程不太可控和可靠，且容易受到人为因素的影响。

1.3

治理模式对比

治理并不是链上化程度越高越好，治理是手段，不是目的。DAO应该根据实际情况选择合适的治理模式。 

链上治理和链下治理各有优缺点。链上治理具有高度的透明度、自动执行以及广泛参与的特点，但可能面临低参与度、激励不足等挑战。链下治理则提供了更多灵活性，便于人工干预和快速响应，但其决策过程可能不够透明且容易受到中心化的影响。 

实际上，许多区块链项目和去中心化应用采用了链上治理和链下治理相结合的方式，以实现更为高效、公平和可持续的治理。例如，一些项目在链上进行核心决策，而将具体实施和具体实施和细节讨论留给链下环境。这样的混合治理模式充分利用了链上和链下治理的优势，既确保了治理过程的透明度和广泛参与，又保留了一定程度的灵活性和人工干预能力。

PART 2

综合 DAO 

Tools 介绍

2.1

Tally（链上治理）

2.1.1 定位：链上治理仪表盘

Tally包括投票仪表板、提案创建向导、实时研究和分析以及教育内容。用户可以在 Tally 上查看发生的事情，然后代理投票权、创建或通过提案来花费 DAO 资金、管理协议或升级智能合约。所有操作都在链上完成。Tally团队认为治理应该像搭乐高一样。在标准形状和大小情况下，构建无限数量的可能。

2.1.2 功能和特色 

Tally 是 Governor 合约的前端，Governor 合约是一种常用的开源治理合约，允许代币持有者使用完全基于链上的投票来控制 DAO。代币持有者使用 Governor 来制定、通过和执行提案。提案可以做任何链上的事情：从资金库中发送资金，更新 DeFi 协议的参数，更改子 DAO 的权限，铸造 NFT，或修改 Governor 本身的规则。 

特色功能： 

Tally API ：使查询有关 Governor DAO 的链上数据变得容易。列出治理者、链上提案和委托账户。使用 API 构建通知、挖掘选民数据或在您的游戏或应用程序中构建治理。

2.1.3 兼容和适应性 

Tally的兼容性很好，支持以太坊、Polygon、Optimism、Arbitrum、Avalanche、BSC、Gnosis和各种测试网络上的DAO治理合约。 

Tally 比较适合成熟业务模块的链上治理，需要自己先用 OpenZeppelin 治理合约模块部署治理合约，然后导入tally，对非专业技术人士不友好。

2.1.4 融资与采用情况 

融资情况：2021年3月，获得融资1.5M，由 lemniscap & 1kx 领投。2021年8月，Tally宣布融资600万美元，Blockchain Capital 和 Placeholder 领投，大约25%的资金来自 The LAO、MetaCartel Ventures、Fire Eyes DAO 等DAO组织，以及与DAO有关联的天使投资者（包括Bankless创始人Ryan Sean Adams）。 

Tally 已被很多大项目采用，大多是业务比较成熟的大项目，包括 arbitrum、sudoswap、nouns、op、uniswap、gitcoin、ens、hop、fei 等。过去的几个月里，由 Alchemy 发起的WEB3 30 评选，经过来自世界各地 web3 爱好者的数千份提名和投票，2023 Web3 30 关于 DAO 和 DAO 工具类评选，Tally 荣获前三名，分别是：Guild、Snapshot 和 Tally。

2.2

Aragon（链上治理）

2.2.1 定位：一站式链上DAO Tool 

无需编码即可启动链上 DAO 治理，及插件式 DAO OS。 

2.2.2 功能和特色 

Aragon App：用户友好的应用程序，无需编码即可启动链上 DAO 治理。该应用程序为您的社区提供了一个简单易用的链上主页，用于创建提案、投票和加入新成员。 

Aragon OSx：通过精益设计，我们的智能合约为我们今天无法想象的组织奠定了基础。插件可以轻松自定义您的治理。使用我们的治理插件在 Aragon OSx 协议上构建 DAO、dApp或任何您可以想象的东西。治理功能插件化，一站式 DAO 治理操作系统！ 

1）构建和管理 DAO 

直接在您的 dApp 中以编程方式部署和运行自定义 DAO。 

2）自定义您的组织 

扩展 DAO 可以使用治理插件（暂未推出）执行的功能。 

3）进化和适应 

我们的权限管理系统授予和撤销权限，因此您的组织可以随着您的需求而发展。 

2.2.3 兼容和适应性

Aragon 是 ico 时代的老项目，中间产品探索和路线发生了较多改变，目前新版产品（Aragon App和Aragon OSx）暂时只兼容以太坊、Polygon 主网和 Polygon/goerli 测试网。 

同样是链上治理，Aragon 是功能全面且易用的链上综合治理工具，对小白和专业人士都适用。 

2.2.4 融资与采用情况

Aragon 在2017年5月进行 ico 代币销售，众筹了27.5万个 eth（按目前以太坊价格，价值约5亿美元）。所以，Aragon 的资金一直很充裕，有时间和资金进行各种产品和路线尝试。

根据Aragon官网，号称：    

5,000 DAOs Created；

+300,000 Community Members；

+$6B InAragon DAOs；

+$200M in Aragon Treasury……

然而，实际新版系统采用相对较少，可能因为2023年2、3月份才推出原因。

2.3

DAOlens（链下治理）

2.3.1 定位：一站式链下DAO Tool

2022 年 1 月创立，旨在帮助企业进行入职、贡献管理等工作来简化 DAO 的运营流程。B2B 商业公司，印度团队。

DAOLens 由 Vikram Aditya 和 Apoorv Nandan 于 2022 年 1 月创立，旨在帮助企业进行入职、贡献管理等工作来简化 DAO 的运营流程。

2.3.2 功能和特色 

DAO Manager 是 DAOLens 的产品名称，有很完善的链下治理基础设施例如：snapshot、bounty、proposal、events、library、space 等，可连接 discord、钱包，权限管理、加密支付等；具有 token 和 nft 门控等。 

初期为了尽量获得用户，DAO 可以不用管 DAO Manager 已有功能，自己设计需求，交给DAOLens，他们在 DAO Manager 中定制实现。DAO Manager 基本版免费，未来靠AI服务、全解决方案包括咨询解决营收。 

2.3.3 兼容和适应性 

因为 DAOLens 是链下综合治理工具，比较中心化的系统，可以接入各种治理小工具，所以它有很好的灵活性，可以支持各种链上代币和nft进行链下辅助治理。适合小白和专业人士，相比链上综合治理工具，更简单和友好，具有很强的适用性，适合各种社群组织和非链上治理 DAO组织。 

2.3.4 融资与采用情况 

Web3 初创公司 DAOLens 宣布完成500万美元融资，Nexus Venture Partners、Better Ventures和 iSeed II领投，Coinbase 董事会成员Gokul Rajaram、Polygon 联合创始人Sandeep Nailwal、Solana 联合创始人 Raj Gokal、Animoca Brands 副总裁 Simon Doherty 和 Tribe Capital 联合创始人 Arjun Sethi 等参投。筹集资金将用于产品开发、人才引进，并推动其工具堆栈的后续推出。 

DAOLens 是 B2B  商业公司，商业拓展能力很强，几乎所有印度人参与的项目，DAOLens 团队都会先拿下，让他们进驻 DAO Manager。包括 Polygon 基金会谈妥，Polygon DAO 已经入驻，还会支持基金会投的项目入驻；Solona基金会也谈妥。

DAOLens 团队也正在游说BanklessDAO （BanklessDAO 的核心成员跟 Polygon DAO 几乎同一拨人）。 

因为起步晚的原因，目前还没有太多有名项目使用它。但DAOLens适用范围广泛，商业前景被很多人看好。

2.4

Zora（链上治理）

2.4.1 定位：为 Nouns DAO 服务的一站式链上治理工具 

    Zora 是一种去中心化协议，任何人都可以在未经许可的情况下进行买卖和创建。创建了一系列工具，可以轻松开始构建。Zora 是为 Nouns DAO 服务的治理工具，允许用户创建自己风格的 Nouns DAO。Nouns 风格的 DAO 具有分布式治理、可组合性和升级性、以及持续融资等特点。 

 2.4.2 功能和产品 

1）Zora堆栈 

    ● Nouns Builder：在几分钟内部署自定义 DAO

     ● 创建：易于为 NFT 项目部署合约 

     ● API：检索任何 ERC-721 上的 NFT 元数据和市场数据 

    ● Zora V3：ZORA 的链上市场功能 

    ● ZDK：用于与 ZORA 的 API 交互的开发工具包 

    ● NFT Hooks：与 ZORA 的 API 交互的最简单方式 

    ● NFT Rendering：轻松展示 NFT 数据的工具

2）模块化架构 

    ZORA V3 使用允许 Zora DAO 部署新市场类型和更新版本的架构。市场模块的每个部署版本都是无需许可的，永远不能更改，但 DAO  可以将模块部署到 Zora 的批准空间中。这有助于开发人员选择他们使用的模块并轻松使用新版本而无需完全迁移（就像我们从 V2 到 V3 所做的那样），并降低了需要花费的总 gas。

3）通过Zora创建具有Nouns特色风格的 DAO

    分配机制：以一致的时间间隔创建NFT可以让社区以健康的速度形成，而不是通过空投来分配治理。 

    可组合和可升级：所有合约都可以单独升级，这意味着 DAO 不是静态的，而是随着时间的推移而演变的。 

    持续融资：DAO 通过随着时间的推移不断发行新的 NFT 来创造可持续的资金。 

 2.4.3 兼容和适应性

    Zora目前仅兼容以太坊主网和 Goerli 测试网。Zora 是为 Nouns DAO 服务的治理工具，应用范围较窄。 

  2.4.4 融资与采用情况

    NFT 交易市场 Zora 以 6 亿美元估值完成 5000 万美元融资，Haun Ventures 领投。Zora 创始人 Katie Haun 曾是 a16z 首位女性普通合伙人，Zora 此前进行了包括 TikTok 相关NFT 以及安迪·沃霍尔艺术基金会 NFT 的拍卖。

    Zora平台自推出以来，已经在 NFT 领域取得了一定的影响力。许多艺术家、创作者和收藏家都在 Zora 上发布和交易 NFT。 

    Zora 的新无代码工具可以更轻松地创建由 NFT 拍卖资助的 DAO，遵循热门项目 Nouns 建立的模型。让创作者更容易使用它，而无需工程师分叉和修改现有代码，使几乎任何人都能够为几乎任何项目创建 NFT 驱动的DAO。购买 NFT 可以让每个所有者成为 DAO 或去中心化自治组织的成员，本质上是一个具有共享金库的代币化社区。随着时间的推移，Nouns 已将其更大的重点放在推进开源知识产权上 ，并利用其充足的资金将 Nouns 图像放入 Bud Light 商业和独立电影中，例如，以及其他社区批准的举措。 

2.5 DAO Tools 小结

    在本报告中，我们分析了四个主要的链上与链下治理工具：Tally、Aragon、DAOLens 和 Zora。每个工具都有其独特的优点和适用场景。根据组织或项目的需求，选择合适的治理工具至关重要。 

    Tally 适用于需要一个成熟且支持多种区块链网络的链上治理工具，但对非专业人士可能不够友好。Aragon 是一个全面且易用的链上治理工具，具有高度可定制性，但仍需要进一步推广和应用。DAOLens 提供了丰富的链下治理基础设施，并与多种工具集成，但目前尚无知名的DAO 或社区采用。Zora 是一个为 Nouns DAO 服务的治理工具，具有创新的治理模式和强大的升级性，但受限于 Nouns DAO 风格和网络支持。 

    建议在选择治理工具时，充分考虑项目的实际需求和目标，以确保选用的工具能够为项目带来最大的价值。

PART 4

关于 DAO Tools 的

未来展望

1）大型 DAO 会选择开发自己的 DAO Tool，小型 DAO 会选择比较通用的 DAO Tool 

随着 DAO 生态的发展，大型 DAO 将根据自身需求定制开发治理工具，以满足其特定业务流程和治理模型。相比之下，小型 DAO 会倾向于使用通用的 DAO 工具，以降低开发成本和提高效率。 

2）市场会涌现各种特定应用的插件化 DAO Tool，供各方集成 

未来将出现更多针对特定应用的插件化 DAO 工具，以满足不同类型 DAO 的需求。这些插件化工具将便于各方集成，提升 DAO Tool 整体的灵活性和可定制性。 

3）大型 DAO Tool 会集成更多外部 DAO Tool 插件 

随着 DAO 生态的不断扩大，大型 DAO 工具将集成更多外部插件，以提供更丰富的功能。大型DAO 工具将为用户提供更加完善的治理体验。 

4）链上治理比例会越来越高，混合治理模式会越来越流行随着区块链技术的进步，链上治理所占比例将逐步提高。

同时，混合治理模式，即结合链上与链下治理的方式，将成为越来越多 DAO 的选择，以提高治理的有效性和适应性。 

5）链上治理数据的增加会推动涌现更多研究和分析链上数据的 DAO Tools 

随着链上治理数据的快速增长，将催生出更多针对链上数据研究和分析的 DAO 工具。这些工具将帮助 DAO 更好地理解治理过程中的数据，为决策提供有力支持。 

6）链上 DAO Tool 的可组合性会推动在不同项目、生态和组织的用户身份和声誉的统合 

未来，链上 DAO 工具的可组合性将促进不同项目、生态和组织之间的用户身份和声誉统一。这一趋势将推动去中心化身份（DID）的发展，为整个区块链生态提供更高效、安全和可靠的身份认证和管理方式。
