#01

比特币L2的

历史演变与现状解析

●

比特币L2技术自2015年Stacks诞生以来备受关注，以其独特的侧链或状态通道技术方案，旨在为比特币网络引入智能合约功能或增强支付领域的应用。然而，在2023年之前，该技术并未得到广泛关注和应用，因为比特币主要被视为价值储存和投资工具，链上生态相对贫瘠。

然而，2023年成为比特币L2技术的转折点。随着主网资产发行协议的爆发，链上交易活跃度显著提升，比特币网络费率一度飙升至600sat/vB。同时，一批优质的一层资产如ordi、蓝盒子、bitmap、nodemonkes等涌现，为比特币生态注入了新的活力。人们开始认识到比特币L2在构建包含DeFi、GameFi、NFT加密艺术等多元化应用方面的巨大潜力。

从职责与使命的角度来看，2023年前后诞生的比特币L2项目也呈现出不同的特点。早期的L2项目主要聚焦于为比特币带来智能合约功能和可扩展性，与以太坊L2的功能相类似。而近年来诞生的L2项目则更加注重于满足实际需求（如质押需求），加强优质资产的流动性和可玩性。尽管这些新项目在某些方面受到传统势力的质疑，但它们无疑为比特币生态的多元化发展注入了新的动力。

展望未来，随着技术的不断完善和市场的逐步扩大，比特币L2有望在加密货币生态中发挥更加重要的作用。我们期待看到更多创新性的L2项目涌现，为比特币生态的繁荣和发展贡献新的力量。

#02

目前比特币

来L2的争议热点

●

当前，比特币L2领域正处于快速发展阶段，各种创新技术和应用不断涌现。然而，与此同时，关于L2的必要性、可信任性和正统性的争议也始终存在。

关于必要性，比特币社区历来秉持保守态度。许多核心开发者对BTC Layer2技术持保留意见，认为其并非必需，更倾向于维持比特币作为纯粹数字黄金的定位。部分极端的比特币原教旨主义者更是排斥所有与DeFi和NFT相关的创新，这与BTC Layer2技术的发展方向背道而驰。此外，比特币的利益相关者众多，却缺乏能够一统江湖的权威声音。与以太坊Rollup的明确发展路径相比，BTC Layer2难以形成强大的合力。即便一些头部协议在TVL方面取得了数十亿美金的成绩，其必要性仍受到部分社区和开发者的质疑。

在可信任性方面，BTC Layer2的解决方案种类繁多，但鲜有能够独领风骚的方案。其中一个重要原因便是对于这些方案可信任性的争议。由于BTC本身不支持智能合约，许多BTC Layer2方案不得不采用多签这种相对原始的方法。用户需将BTC资产存入多签地址，随后在Layer2上生成对应资产。这种方式在资金安全性方面存在隐患，对用户的可信任性构成了巨大挑战，因此采用类似方案的BTC Layer2项目一直饱受争议。

至于正统性，今年老牌的比特币资讯媒体《Bitcoin Magazine》在其官方发文中明确提出了BTC Layer2的标准。这些标准包括：使用比特币作为原生资产；使用比特币作为结算机制来强制执行交易；展示对比特币的功能依赖性。按照这些标准来看，当前市场上的一些热门项目似乎并不符合BTC Layer2的定义。然而，经过各方解读，这篇文章更多地体现了意识形态之争。西方加密圈试图通过重新制定标准来确立某些项目的正统地位，从而重塑BTC Layer2的叙事。

综上所述，BTC Layer2赛道尚处于发展初期，各种争议表明该领域仍充满大量机遇。若该赛道缺乏争议且高度一致，则可能意味着其已进入成熟期，对于参与者而言，早期红利将大打折扣。

注1：ZK-Rollup。ZK-Rollup 是零知识证明和整合交易信息的有机结合。零知识证明用以保证整合交易过程中信息的准确性，完整性和上传一层网络时的安全性。

2.BEVM

项目基础信息：

区块链桥，跨链桥，质押

（1）官方推特：@BTClayer2

（2）官方DC：

https://discord.com/invite/gRJ72RcrNK

项目技术介绍：

（1）资产从 Bitcoin blockchain 去向 BEVM

BEVM通过部署 轻节点 同步完整的比特币主网数据，保证去中心化和数据的正确性和有效性，实现在用户资产需要跨链时同步相关的交易数据和交易证明，等待 BEVM 的共识机制验证后完成单向的跨链交互。

（2）资产从 BEVM 回到 Bitcoin blockchain

BEVM 采用 Taproot 技术 和 PoS共识机制 来实现将 BEVM 的链上资产和数据以去中心化的方式跨链回到比特币主网。

BEVM的Taproot 共识由三部分组成：第一部分：Taproot技术，包括Musig2、Schnorr签名、MAST等。第二部分：由比特币SPV组成的BFT PoS网络。第三部分：通过信号协议形成的阈值节点通信。

（3）实现核心功能的框架

BEVM 采用 Substate 框架 来构建区块链，实现 EVM 兼容以及将比特币作为 Gas Fee 的底层逻辑。

（4）整体技术框架

BEVM 的整体技术框架自下而上分为四层，第一层是介于比特币主网与BEVM 链的去中心化交互层，第二层是结合 Aura 和Grandpa 的共识层，第三层是兼容EVM 的智能合约层，第四层是支持编程语言Solidity 建构 DApp 的应用层。

注1：轻节点。是指一种连接到区块链的设备或软件，它与全节点和矿工节点相比，具有较小的存储空间和带宽需求，只保存区块头信息，不保存完整的区块链历史记录。

注2：区块头信息。在区块链网络中，区块由区块头和区块体两部分组成。区块头包含丰富的信息，包括前一个区块的哈希值、当前区块的哈希值和时间戳，交易执行情况信息和矿工工作信息等。

注3：Taproot。 Taproot 是指通过多方同时签名改善交易的隐私并降低交易费用的技术。

注4：权益证明 PoS。权益证明 PoS共识机制 是一种通过质押数字资产产生新区块的方式，可防止重复计算问题，从而显著降低能耗。

注5：Substate 框架。 Substate 框架提供了一种模块化和可扩展的方法来构建区块链应用程序。

注6：Aura 和 Grandpa。 Aura是一种共识算法，用于创建区块和轮流出块。它是一种升级版的DPOS（Delegated Proof of Stake）共识算法，具有高TPS（Transactions Per Second）的特点。Grandpa是一种权限，用于最终确定区块。它使用最长链规则来投票，并与Aura共同使用。Grandpa是一种用于确定区块的权限，与Aura共同构成了区块链的共识机制。

注7：DApp。 DApp是指去中心化应用（Decentralized Application），它是一种运行在区块链上的应用程序。

10. Map Protocol

项目基础信息：

（1）官方推特：@MapProtocol

（2）官方TG：

 https://t.me/MAPprotocol/

项目技术介绍：

MAP Protocol 由三部分组成：

1.协议层 Protocol Layer —— 全链通讯验证的核心

MAP Protocol 协议层是最底层，也是全链通讯验证的核心，负责跨链验证。这一层由 MAP Replay Chain 、各链布署的轻节点，以及链间消息传递程序 Maintainer 构成。MAP Relay Chain 虚拟机层以预编译合约的方式成功将各大 L1 签名算法、 hash 算法和 Merkle Tree 证明内置，让 MAP Relay Chain 就像精通各国语言的超级语言机器，通过 MAP Relay Chain 链与链之间就能相互沟通，这为各链互通打下了同构基础。

各链布署的轻节点具有独立自验证和即时最终性保证的特点，基于 MAP Relay Chain 的同构基础，轻节点交叉验证网络可以拥有相同的数据语言，能以智能合约的形式轻松地部署到任何对应的 L1 上，继而进行去中心化的跨链有效性验证。

Maintainer 是独立的链间消息传递程序，负责更新轻节点的最新状态，将各个链的共识层区块头（Validator 签名）信息，以交易形式写入目标链上的起源链的轻节点智能合约中，以此保障目标链上的起源链的轻节点与起源链的 Validator 信息一致。

拥有多种预编译合约植入的 MAP Relay Chain 在整个业界是独一无二的存在，相比其他轻节点跨链技术方案，MAP Protocol 可以覆盖所有 L1。结合其独特的跨链通信组件，MAP Protocol 将会大大减轻数据跨链和资产全链自由流动的阻碍。

2.MOS 全链服务层 MAP Omnichain as a Service Layer —— 面向 dApp 开发者

 MOS 层示意图

MOS 层是第二层，类似于面向安卓生态的 Google Mobile Service，为 dApp 开发者提供全链开发服务。该层拥有部署于各个区块链上的跨链资产锁定智能合约和链间消息传递组件 Messenger，开发者可直接利用这一层建立全链应用场景，也可根据自己的需求进行进一步编译。这一层的智能合约都是通过 CertiK 审计的开源组件，dApp 开发者可以直接使用，无需担心安全与开发成本的问题，由此节省全链的开发和学习成本。

3.全链应用层 Omnichain Application Layer —— 拓宽全链 dApp 生态

全链应用层与 dApp 全链生态的发展有关，MOS 层的全链服务能让 dApp 实现互操作性，同时协议层数据资产的验证网络可以推动 dApp 不断拓展其现有生态，由此实现各链互通的全链生态。

 MAP Protocol 全链数据流通示意图

以去中心化衍生品和合成资产为例，目前二者受制于其他链上资产价格、数量的情况，通过链下预言机无法获取精准、及时的资产数据信息，所以流动性和用户体验都欠佳。虽然多链布署可以解决这一问题，但这一过程耗时费力，更会增加不必要的开发成本。而如果在 MAP Replay Chain 上完成布署，去中心化衍生品和合成资产就可以从 MAP Protocol 链上预言机上获得精准的多链数据，不再受制于数据流动的阻碍，从而轻松实现资产的全链流动。

类似应用场景还有全链 DID、全链借贷、全链 Swap、全链 GameFi、全链 DAO 治理、全链代币及全链 NFT，无论 dApp 的主业务合约部署在哪个 L1 上，通过 MAP Protocol 开发者即可轻松构建有能力覆盖所有链用户和资产的全链应用程序。
